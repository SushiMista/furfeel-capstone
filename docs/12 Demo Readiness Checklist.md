# Demo Readiness Checklist

## Before Any Demo
- [ ] Confirm Supabase project is accessible.
- [ ] Confirm test users can log in.
- [ ] Confirm sample dog exists.
- [ ] Confirm sample device exists.
- [ ] Confirm telemetry records exist.
- [ ] Confirm stress classification records exist.
- [ ] Confirm alerts exist.

## Demo Flow
- [ ] Show project goal.
- [ ] Show telemetry being received.
- [ ] Show Supabase record.
- [ ] Show mobile owner status.
- [ ] Show web clinic dashboard.
- [ ] Show stress classification.
- [ ] Show alert.
- [ ] Explain videos/media as supplementary.
- [ ] Explain rule-based MVP and future Random Forest.

## Backup Plan
- [ ] Save screenshots of every demo step.
- [ ] Save sample JSON payload.
- [ ] Save recorded screen demo if possible.
- [ ] Prepare explanation for hardware unavailability.

## Development Trick
Keep one “golden demo dog” with stable, known sample data. Do not use random messy test data for defense demos.
