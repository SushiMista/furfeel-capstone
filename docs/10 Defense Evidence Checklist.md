# Defense Evidence Checklist

## Architecture Evidence
- [ ] System architecture diagram.
- [ ] Data flow diagram.
- [ ] Supabase schema screenshot.
- [ ] API or Edge Function examples.
- [ ] ESP32/simulator payload example.

## Telemetry Evidence
- [ ] Sample telemetry JSON.
- [ ] Supabase telemetry records.
- [ ] Latest reading displayed in Flutter.
- [ ] Latest reading displayed in React.
- [ ] Device offline or missing-data behavior.

## Classification Evidence
- [ ] Rule table for stress levels.
- [ ] Sample Calm classification.
- [ ] Sample Mild Stress classification.
- [ ] Sample Moderate Stress classification.
- [ ] Sample High Stress classification.
- [ ] Alert generated from Moderate or High Stress.

## UI Evidence
- [ ] Flutter login/status screenshots.
- [ ] Flutter alert screenshots.
- [ ] React dashboard screenshot.
- [ ] React dog detail screenshot.
- [ ] React alert queue screenshot.
- [ ] Media submission/review screenshots if included.

## Evaluation Evidence
- [ ] ISO/IEC 25010 questionnaire or rubric.
- [ ] Performance response time result.
- [ ] Usability feedback.
- [ ] Reliability test notes.
- [ ] Security/RLS test notes.

## Defense Safety Net
- [ ] Backup screenshots in case live demo fails.
- [ ] Backup sample data in Supabase.
- [ ] Clear limitations slide.
- [ ] Future work slide: expert-labeled data and Random Forest.
