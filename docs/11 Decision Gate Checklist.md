# Decision Gate Checklist

Use this before accepting a major project decision.

## Decision Quality
- [ ] The decision has an owner.
- [ ] The decision has a reason.
- [ ] The decision has affected notes linked.
- [ ] Alternatives were considered briefly.
- [ ] The impact on MVP scope is known.
- [ ] The impact on defense claims is known.

## Record Keeping
- [ ] Add the decision to [[02 Architecture Decisions]] if technical.
- [ ] Add the decision to [[Product Requirements Index]] if product-facing.
- [ ] Add the decision to [[15 Open Technical Questions]] if it resolves a question.
- [ ] Add follow-up tasks to [[20 Development Backlog]].

## Development Trick
Use “accepted for MVP” when a choice is practical now but may change later. This keeps the team moving without pretending the decision is permanent.
