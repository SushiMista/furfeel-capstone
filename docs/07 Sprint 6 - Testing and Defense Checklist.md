# Sprint 6 - Testing and Defense Checklist

## Goal
Turn the MVP into a defensible capstone prototype.

## Testing
- [ ] Test Flutter owner flow.
- [ ] Test React clinic flow.
- [ ] Test telemetry intake.
- [ ] Test rule-based classification examples.
- [ ] Test alert creation.
- [ ] Test alert acknowledgement.
- [ ] Test Supabase Row Level Security.
- [ ] Test media upload/review if included.

## Evaluation
- [ ] Map features to ISO/IEC 25010 functional suitability.
- [ ] Measure dashboard response time.
- [ ] Measure app response behavior.
- [ ] Evaluate usability with test users.
- [ ] Document reliability limitations.
- [ ] Document security controls.

## Defense Prep
- [ ] Prepare demo script.
- [ ] Prepare backup screenshots.
- [ ] Prepare sample telemetry records.
- [ ] Prepare classification examples.
- [ ] Prepare architecture diagram.
- [ ] Prepare limitations and future work.

## Exit Criteria
- [ ] The team can run a full demo without improvising.
- [ ] Evidence exists for every claimed feature.
- [ ] Known limitations are documented honestly.
